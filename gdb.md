# GDB 调试实用手册

## 1. 编译准备

在使用 GDB 调试程序之前，必须在编译时加入**调试符号**。

- **C/C++ 编译**：
  ```bash
  gcc -g -o program program.c
  ```
- **NASM 汇编编译**（添加 DWARF 格式的调试信息）：
  ```bash
  nasm -f elf64 -g -F dwarf -o first.o first.s
  ld -m elf_x86_64 -o first first.o
  ```

## 2. 启动与退出

| 命令                | 简写 | 说明                          |
| :------------------ | :--- | :---------------------------- |
| `gdb <file>`        | -    | 启动 GDB 并加载可执行文件     |
| `gdb <file> <core>` | -    | 调试崩溃产生的 core dump 文件 |
| `quit`              | `q`  | 退出 GDB                      |

---

## 3. 运行与断点 (Breakpoint)

在调试过程中，断点用于让程序暂停在特定位置。

| 命令                 | 简写  | 示例                   | 说明                                         |
| :------------------- | :---- | :--------------------- | :------------------------------------------- |
| `run`                | `r`   | `run`                  | 开始运行程序（直到遇到断点或结束）           |
| `break`              | `b`   | `b main` 或 `b _start` | 在函数或入口处设置断点                       |
| -                    | -     | `b 15`                 | 在当前文件的第 15 行设置断点                 |
| -                    | -     | `b *0x401000`          | 在内存地址 `0x401000` 处设置断点（汇编常用） |
| `info breakpoints`   | `i b` | `i b`                  | 查看所有断点信息                             |
| `delete`             | `d`   | `d 1`                  | 删除编号为 1 的断点                          |
| `disable` / `enable` | -     | `disable 1`            | 禁用/启用编号为 1 的断点                     |

## 4. 程序执行控制

对于汇编语言调试，**单步执行指令**（`si` 和 `ni`）比单步执行源码行（`s` 和 `n`）更为常用。

| 命令       | 简写 | 说明                                             |
| :--------- | :--- | :----------------------------------------------- |
| `continue` | `c`  | 继续运行程序，直到遇到下一个断点                 |
| `next`     | `n`  | 单步跟踪（逐行），不进入函数内部（Source-level） |
| `step`     | `s`  | 单步跟踪（逐行），进入函数内部（Source-level）   |
| `nexti`    | `ni` | **单步执行一条汇编指令**，不进入子程序调用       |
| `stepi`    | `si` | **单步执行一条汇编指令**，进入子程序内部         |
| `finish`   | -    | 运行直到当前函数返回                             |

## 5. 数据查看与修改

### 5.1 查看寄存器（汇编核心）

| 命令             | 简写  | 示例          | 说明                                  |
| :--------------- | :---- | :------------ | :------------------------------------ |
| `info registers` | `i r` | `i r`         | 查看所有通用寄存器的值                |
| -                | -     | `i r rax rbx` | 查看特定寄存器 `rax` 和 `rbx` 的值    |
| `print`          | `p`   | `p /x $rax`   | 以十六进制（`/x`）格式输出 `rax` 的值 |

### 5.2 查看内存 (Examine)

格式：`x/<n><f><u> <addr>`

- `n`：要显示的单位数量
- `f`：显示格式（`x` 十六进制，`d` 十进制，`i` 汇编指令，`s` 字符串）
- `u`：单位大小（`b` 字节，`h` 双字节，`w` 四字节，`g` 八字节）

| 示例命令     | 说明                                                               |
| :----------- | :----------------------------------------------------------------- |
| `x/10i $pc`  | 从当前指令指针（PC）处开始，反汇编显示 10 条指令                   |
| `x/4xg $rsp` | 以十六进制、8字节（Quad-word）为单位显示栈顶 `rsp` 指向的 4 个数值 |
| `x/s &var`   | 显示变量 `var` 处的字符串                                          |

### 5.3 修改数据

```text
(gdb) set $rax = 0x10       # 将寄存器 rax 的值改为 0x10
(gdb) set var x = 5         # 将变量 x 的值改为 5
```

## 6. TUI (文本用户界面) 模式

TUI 可以在终端中分屏显示源代码、汇编代码和寄存器状态。

- **常用视图命令**：
  - `layout asm`：显示汇编代码窗口。
  - `layout regs`：显示寄存器和汇编代码。
  - `layout split`：同时显示源码和汇编代码（如果可用）。
  - `layout src`：显示源码窗口。
- **常用快捷键**：
  - `Ctrl + X` 紧接着按 `A`：开启或关闭 TUI 模式。
  - `Ctrl + L`：当屏幕由于程序输出变乱时，刷新屏幕。
  - `focus cmd`：将键盘焦点切换回底部的命令行，方便输入命令或上下翻阅历史。

## 7. 栈帧查看 (Call Stack)

当程序在函数中暂停时，可以用以下命令查看调用栈：

| 命令          | 简写 | 说明                           |
| :------------ | :--- | :----------------------------- |
| `backtrace`   | `bt` | 显示当前调用栈（函数调用关系） |
| `frame <num>` | `f`  | 切换到指定编号的栈帧           |
| `info locals` | -    | 查看当前栈帧中的局部变量       |
